## 🛠 Documentation Tools

### Storybook
Interactive component documentation and development environment.

```bash
# Start Storybook server
npm run storybook

# Build static Storybook site
npm run build-storybook
```

Visit `http://localhost:6006` to browse component documentation.

### JSDoc Comments
All components and functions include JSDoc comments for inline documentation.

## 📝 Documentation Standards

### When to Update Documentation

Update documentation when:
- ✅ Adding new features or components
- ✅ Changing existing functionality
- ✅ Modifying API endpoints or data contracts
- ✅ Updating project structure or architecture
- ✅ Changing development workflow or tools

### Documentation Checklist

For every significant change:
- [ ] Update README.md if it affects setup or usage
- [ ] Update architecture.md for structural changes
- [ ] Update components-guidelines.md for new patterns
- [ ] Update api.md for API changes
- [ ] Add/update JSDoc comments in code
- [ ] Create/update Storybook stories for components
- [ ] Update contributing.md if workflow changes
